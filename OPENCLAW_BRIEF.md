# OpenClaw Brief

## Goal
Build and submit a polished project for `Smoke Test Hackathon`.

## Inputs
- Rules: `hackathon.json`
- Submission fields: `devpost/answers.json`
- Acceptance gates: `OPENCLAW_ACCEPTANCE.json`

## Required outcomes
1. Working app in `app/`
2. Tests in `tests/`
3. Public GitHub repo URL recorded in `hackathon.json`
4. Submission copy completed under `devpost/`
5. Screenshots in `artifacts/screenshots/`
6. Verification notes in `artifacts/verification.md`

## Constraints
- Respect the normalized rules in `hackathon.json`.
- Keep generated assets and code inside this workspace.
- Do not claim completion until acceptance checks pass.
